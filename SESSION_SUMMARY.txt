ED-721 프로젝트 세션 요약
=========================

이 파일은 초기 세션(SPEC 분석 + 단말정보 요청 구현)의 메모로 시작되었으나,
이후 단계적 단위테스트/실거래 검증/리팩토링/NAK 재전송 정책 도입을 거치며
정보가 크게 보강되었습니다. **현재 상태와 운영 가이드는 다음 문서를 보세요:**

- docs/REPORT.md           작업 종합 보고서 (모든 Phase 결과 + 운영 가이드)
- docs/00_overview.md      전체 가이드와 환경 정보
- docs/01_phase1_protocol.md  프로토콜 단위 테스트 (50개)
- docs/02_phase2_hw.md     하드웨어 비파괴 검증
- docs/03_phase3_realtxn.md  실거래 시나리오 + R09 핵심 발견
- docs/04_phase4_codefix.md  응답 판정 헬퍼
- docs/05_phase3_extra_image.md  이미지 다운로드 검증
- docs/06_phase3_validation_crc.md  입력검증 / CRC / 연속송신
- docs/07_phase4_refactor_nak_retry.md  리팩토링 + NAK 재전송 정책

핵심 결과 (간략)
- pytest 87/87 통과 (프로토콜 50 + 응답 판정 25 + NAK 재전송 12)
- 코드 35% 감소 (1300 → 846 LOC)
- **운영상 가장 중요한 발견**: 취소 명령 S12에 넣을 승인번호는 S07 이 아니라 R09.
  S07 은 단말 placeholder(이 단말은 항상 949094 반환). RCD=0x00 만으로 취소 성공
  판정 금지 — R17 부재 + R19 정상값 + R09 존재 조합으로 검증해야 함.
- NAK 자동 재전송: 거래 명령(FB/14/04)은 NEVER 강제, 그 외 SAFE 3회.
- 이 단말은 LA(직전거래) 미지원 (등록 상태와 무관 항상 FAIL).

자주 쓰는 명령은 ED721_CLI_Commands.txt 참고.
