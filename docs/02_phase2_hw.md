# Phase 2 — 하드웨어 in-loop, 비파괴 테스트

실행: 2026-04-29, `/dev/ttyUSB0`, 단말 `192.168.0.224` (K03 v0023, TID 8016366)

## 테스트 시나리오 & 결과

### 1. info 반복 + CNT 증가
| 송신 CNT | 응답 CNT |
|---------|---------|
| 0x0A | 0x02 |
| 0x0B | 0x03 |

✅ 정상. **핵심 발견**: 단말은 **자체 응답 카운터**를 가지며, 응답을 보낼 때마다 +1. 송신 CNT와 무관.

### 2. init (FB/14/01)
- TX: `02 00 06 01 FB 14 01 03 65 B9`
- RX: `06 06 06` (ACK only)
- ✅ SPEC상 응답 전문 없음, ACK만 수신 = 정상

### 3. display (FB/14/03)
- DATA: `"TEST LINE 1|TEST LINE 2|TEST LINE 3|TEST LINE 4"` (`|` = 줄바꿈)
- TX(부분): `02 00 35 01 FB 14 03 ... 03 50 8E`
- RX: ACK only
- ✅ **단말 화면에 4줄 메시지 정상 표시 (육안 확인)**

### 4. sound (FB/14/13)
| 파일명 | 결과 |
|--------|------|
| `beep.wav` | ✅ 삑 소리 재생 |
| error.wav, ok.wav, alarm.wav, success.wav, fail.wav, cancel.wav, card.wav, 1.wav, 2.wav | ❌ ACK만 받고 무음 (파일 부재 추정) |

**관찰**: 파일이 없어도 단말은 NAK 안 주고 ACK만 보냄. 응답 본문이 없는 명령이라 성공/실패 구분 불가능. **운영상 주의 필요**.

### 5. RFID (FB/01/06)
- TX: `02 00 06 01 FB 01 06 03 DE F4`

**5-A. 타임아웃/사용자 취소**
- 단말 화면: "카드를 대주세요" 표시 → 카드 미태그 → 자동 취소
- 부가 동작: **"결제 취소" 화면 + "죄송합니다" 음성 안내** 자동 출력
- RX: `06 06 06 02 00 0F 04 FB 01 06 FF 43 41 4E 43 45 4C 45 44 03 A8 A4`
- 파싱: RCD=FF, DATA = `"CANCELED"` (8B ASCII)

**5-B. 정상 태그**
- 카드 태그 후 응답: `06 06 06 02 00 0B 05 FB 01 06 00 07 AF C6 81 03 5D 4C`
- 파싱: RCD=00, SNO = `07AFC681` (4B binary)

✅ SPEC 일치. 실패시 사용자 친화적 안내(화면+음성) 자동 처리됨.

### 6. 잘못된 framing → NAK
- 송신: `len_mode=0` 으로 info 요청 (`02 00 04 01 FB 14 02 03 50 EB`)
- 수신: `15 15 15` (NAK triple)
- ✅ 단말이 LEN 오류를 정확히 NAK으로 거부함. NAK 감지 코드 동작 확인.

## 신규 발견사항 정리 (Phase 3 진입 전 반영 필요)

1. **단말 자체 CNT**: 응답 패킷마다 +1. 송신 CNT와는 별개. → 테스트에서 송신 CNT로 응답 매칭하면 안 됨.
2. **RFID 실패 시 단말 자동 UI**: "결제 취소" + "죄송합니다" 음성. POS 측에서 별도 안내 추가 시 중복 주의.
3. **sound 파일 부재 시 silent ACK**: 파일 존재 검증 불가. 사용 가능한 wav는 현재 `beep.wav`만 확인.
4. **응답 없는 명령**(init/display/sound): ACK만으로 성공 판정해야 함. CLI는 현재 "응답 파싱 실패"로 exit 2 반환 → Phase 4에서 ACK-only 케이스 정상 처리 개선 필요.
